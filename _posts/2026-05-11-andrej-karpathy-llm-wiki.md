## 1. 개요: 기존 RAG(검색)의 한계와 새로운 패러다임

우리가 흔히 사용하는 기존의 **RAG(Retrieval-Augmented Generation, 검색 증강 생성)** 방식은 근본적으로 '검색'에 의존합니다. 사용자가 질문할 때마다 관련 문서를 벡터 데이터베이스에서 검색해 AI에 프롬프트로 꽂아주고 답변을 생성한 뒤, 시스템은 모든 대화 컨텍스트를 잊어버립니다. 

안드레 카파시(Andrej Karpathy)는 이러한 방식이 비효율적이고 파편화되어 있다고 말합니다. 대신 그는 **지식을 프로그램 소스 코드처럼 관리하고, AI 에이전트를 통해 이를 체계적인 '위키(Wiki)'로 컴파일(Compile)하는 방식**을 제안했습니다. 이것이 바로 **LLM Wiki** 패러다임입니다.

---

## 2. LLM Wiki의 핵심 아키텍처

카파시는 이 구조를 소프트웨어 개발 환경에 빗대어 설명합니다.

> **"Obsidian은 IDE이고, AI 에이전트(Claude Code 등)는 프로그래머이며, Wiki 마크다운 파일들은 소스 코드(코드베이스)이다."**

- **소스 폴더 (Source Folder):** 정리가 안 된 오리지널 자료들(논문 PDF, 북마크 링크, 메모 스크랩 등). 이는 불변(Immutable) 상태로 유지되는 날 것의 정보입니다.
- **AI 에이전트 (Programmer):** 사람이 소스 폴더에 새로운 파일을 넣으면, 백그라운드에서 AI 에이전트가 이를 읽어 분석하고 핵심을 요약합니다.
- **위키 폴더 (Codebase):** 에이전트가 컴파일하여 만들어낸 상호 연결된 마크다운 파일들의 집합(`[[wikilinks]]` 구조). 사용자는 이 정돈된 위키를 편하게 읽고 탐색합니다.

## 3. RAG vs LLM Wiki 비교

| 구분 | 일반적인 RAG 환경 | LLM Wiki 워크플로우 |
| :---: | :--- | :--- |
| **정보 처리 방식** | 질문할 때마다 실시간으로 관련 문서를 검색 | 새로운 소스가 추가될 때마다 즉시 지식 베이스에 병합 |
| **지식의 형태** | 쪼개진 원본 텍스트 조각들 (Vector Chunks) | 사람이 읽기 쉽게 정돈되고 하이퍼링크된 마크다운 위키 |
| **에이전트의 역할** | 단순히 검색된 결과를 조합해 답변 생성 | 지식 간의 모순을 해결하고, 분류 체계를 동적 갱신 |
| **축적성** | 매 질문마다 일회성으로 소비 | 데이터가 쌓일수록 상호 연결 관계가 단단해짐 (지식의 복리 효과) |

## 4. LLM Wiki는 어떻게 작동하는가?

1. **스키마 정의 및 지침 작성:**
   개인용 컴퓨터의 특정 폴더 안에 AI 에이전트가 지켜야 할 규칙 문서(예: `CLAUDE.md`)를 작성합니다. 여기에는 위키 페이지의 템플릿 양식, 카테고리 구성 규칙, 링크 작성 규칙 등이 포함됩니다.
2. **신규 자료 투입:**
   사용자가 읽고 싶은 최신 논문 PDF나 웹 기사 텍스트를 `/sources` 폴더에 넣습니다.
3. **에이전트 컴파일:**
   AI 에이전트(예: Claude Code)를 실행하면, 에이전트가 변경 감지를 통해 새 소스를 읽고 관련 기존 위키 페이지를 업데이트하거나 새로운 위키 페이지를 만들어 기존 페이지들과 하이퍼링크(`[[링크]]`)로 자동 연결합니다.
4. **휴먼 피드백 (Human-in-the-Loop):**
   사용자는 Obsidian과 같은 마크다운 편집기(IDE)로 컴파일된 위키를 읽으며, 잘못된 요약이나 분류가 있다면 수동으로 고칩니다. AI 에이전트는 사람이 수정한 최종 결과물을 다시 소스로 학습하여 다음 작업에 반영합니다.

## 5. 마치며: '읽기 전용' 인간과 '쓰기 전용' AI

지식 관리의 패러다임은 **"내가 직접 쓰고 정리하던 시대"**에서 **"AI가 고도로 정리해 놓은 결과물을 내가 검토하고 소비하는 시대"**로 전환되고 있습니다. 

안드레 카파시의 LLM Wiki는 단순 검색 도구에 의존하던 AI 활용법을 **능동적인 개인 지식 컴파일러**로 한 단계 격상시키는 실전 디자인 패턴입니다. 늘어나는 북마크와 읽지 않는 PDF 더미 속에서 지식의 부채를 겪고 있다면, 오늘 당장 나만의 LLM Wiki 폴더를 구성해 보시기 바랍니다.
