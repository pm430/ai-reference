---

## 1. 시작하기

- **설치** — npm 또는 네이티브 설치 후 터미널에서 `claude` 한 줄로 실행합니다. 설치 방법은 {% post_url 2026-05-07-ai-cli-tools-installation %}에 OS별로 정리돼 있습니다.
- **실행 위치** — 프로젝트 루트에서 실행하면 Claude가 해당 디렉터리를 작업 기준으로 삼습니다.
- **어디서 쓰나** — 터미널 CLI뿐 아니라 데스크톱 앱, VS Code·JetBrains 확장, 웹(claude.ai/code)에서도 동작합니다.

---

## 2. 컨텍스트 관리가 절반이다

Claude Code를 잘 쓰는 사람과 못 쓰는 사람의 차이는 대부분 **컨텍스트 관리**에서 갈립니다.

- **`CLAUDE.md`** — 프로젝트 루트에 두면 매 세션 자동으로 읽힙니다. 코딩 규칙, 빌드·테스트 명령, 디렉터리 구조 같은 "매번 설명하기 귀찮은 것"을 여기에 적어둡니다.
- **`/clear`** — 주제가 바뀌면 대화를 비웁니다. 이전 작업의 잔여 컨텍스트가 새 작업을 오염시키는 것을 막습니다.
- **`/compact`** — 긴 대화를 요약해 컨텍스트를 압축합니다. 맥락은 유지하면서 토큰을 아낍니다.
- **파일 참조** — `@경로/파일` 로 특정 파일을 콕 집어 컨텍스트에 넣을 수 있습니다.
- **이미지·로그 붙여넣기** — 스크린샷이나 에러 로그를 그대로 붙여 넣으면 그것을 근거로 작업합니다.

> 컨텍스트는 무한하지 않습니다. "필요한 것만, 필요할 때" 넣는 습관이 결과 품질을 좌우합니다.

---

## 3. 안전하게 일 시키기

- **Plan 모드** — 바로 코드를 고치기 전에 계획부터 세우게 합니다. 큰 변경일수록 계획을 먼저 검토하면 헛작업이 줄어듭니다.
- **권한(permissions)** — 파일 쓰기, 명령 실행 등은 권한 모드에 따라 확인을 받습니다. 민감한 동작을 자동 승인하지 않도록 모드를 점검하세요.
- **되돌리기 쉬운 단위로** — 한 번에 너무 많이 시키기보다, 검토 가능한 작은 단위로 나눠 진행하면 통제하기 쉽습니다.

---

## 4. 확장 기능 — 단순 채팅을 넘어서

Claude Code의 진짜 힘은 확장에 있습니다. 각 기능의 개념은 {% post_url 2026-05-07-claude-skill-subagent-command %}에서 더 자세히 다룹니다.

| 기능 | 한 줄 정의 | 언제 쓰나 |
|---|---|---|
| **Skills** | 반복 워크플로우를 묶은 재사용 단위 | 같은 절차를 자주 반복할 때 |
| **Subagents** | 독립 컨텍스트로 도는 전문 하위 에이전트 | 메인 대화를 오염시키지 않고 병렬·전문 작업 |
| **Slash Commands** | `/`로 호출하는 빠른 동작 | `/clear`, `/compact`, 커스텀 명령 |
| **MCP 서버** | 외부 시스템(DB·문서·브라우저) 연결 | Claude가 코드 밖 정보에 접근해야 할 때 |
| **Hooks** | 도구 이벤트에 자동 실행되는 동작 | 자동 포맷·lint, 민감 파일 차단 |
| **Plugins** | 위 기능들을 묶어 배포하는 패키지 | 마켓플레이스에서 한 번에 설치 |

MCP가 무엇이고 왜 중요한지는 {% post_url 2026-05-18-mcp-the-future-of-ai-tooling %}를 참고하세요.

---

## 5. 무엇부터 설정할지 모르겠다면

확장 기능이 많아 막막하다면, 코드베이스를 분석해 **카테고리별 자동화를 추천해주는** 공식 플러그인이 출발점을 잡아줍니다.

```bash
/plugin install claude-code-setup@claude-plugins-official
```

읽기 전용으로 코드베이스를 스캔한 뒤, MCP·Skills·Hooks·Subagents·Slash Commands 각 영역에서 효과 큰 것을 골라줍니다. 자세한 내용은 {% post_url 2026-06-08-claude-code-setup-guide %}에 정리했습니다.

## 마무리

Claude Code는 "AI에게 코드 좀 고쳐줘"라고 던지는 채팅창이 아니라, **컨텍스트를 설계하고 워크플로우를 자동화하는 개발 환경**입니다. 영상으로 큰 그림을 잡았다면, 이제 `CLAUDE.md`부터 한 줄 적어보는 것이 가장 빠른 첫걸음입니다.
